# 船舱装载优化项目需求规范

---

## 一、项目背景

某仓库需要通过船舱向目标仓库补充货物。船舱容量有限（2000t/2000m³），需要在保证库存消耗平衡的条件下，尽可能填满船舱。

---

## 二、输入参数

### 2.1 船舱容量

| 参数 | 数值 | 单位 |
|:-----|:-----|:-----|
| 重量容量 | 2000 | t (吨) |
| 体积容量 | 2000 | m³ (立方米) |

### 2.2 物品数据

| 物品代码 | 现有库存(I) | 每日消耗(D) | 单位重量(w) | 单位体积(v) |
|:---------|:------------|:------------|:------------|:------------|
| GRN | 875 | 187.5 | 0.9 | 1.0 |
| NUT | 965 | 187.5 | 0.9 | 1.0 |
| MUS | 853 | 187.5 | 0.8 | 1.0 |
| DW | 224 | 44.8 | 0.1 | 0.1 |
| OVE | 28 | 5.6 | 0.02 | 0.025 |
| COF | 28 | 5.6 | 0.1 | 0.1 |
| PWO | 11 | 2.24 | 0.05 | 0.05 |

---

## 三、核心规则

### 3.1 平衡条件

> 补充后，各物品的库存天数必须相同

$$\text{库存天数} = \frac{\text{现有库存} + \text{装载量}}{\text{每日消耗}}$$

- 所有物品的库存天数 **必须相等**
- 目标仓库的每日消耗量等于表格中的"每日消耗"列
- 目标仓库库存**没有上限**

### 3.2 填充率计算规则

> 按重量和体积中**占用比例较高的那个**计算填充率

$$\text{填充率} = \max\left(\frac{\text{总重量}}{C_w}, \frac{\text{总体积}}{C_v}\right)$$

### 3.3 装载约束

- 装载量不能为负数（不能从库存中取走物品）
- 重量不超过 2000t
- 体积不超过 2000m³

---

## 四、输出要求

### 4.1 必须输出

| 输出项 | 说明 |
|:-------|:-----|
| 最优平衡天数 | 所有物品统一的库存维持天数 |
| 各物品装载量 | 每种物品的装载数量 |
| 总重量 | 装载物品的重量总和 |
| 总体积 | 装载物品的体积总和 |
| 填充率 | 按瓶颈资源计算的填充率 |

### 4.2 输出格式示例

```
最优平衡天数: 8.12 天
填充率: 99.9%

物品装载明细:
| 物品代码 | 装载数量 | 重量(t) | 体积(m³) |
|----------|----------|---------|----------|
| GRN      | 649      | 584.1   | 649      |
| NUT      | 611      | 549.9   | 611      |
| ...      | ...      | ...     | ...      |

船舱利用:
| 指标     | 数值   | 容量   | 填充率 |
|----------|--------|--------|--------|
| 总重量   | 1691 t | 2000 t | 84.5%  |
| 总体积   | 1998 m³| 2000 m³| 99.9%  |
```

---

## 五、验收标准

| 序号 | 标准 | 说明 |
|:-----|:-----|:-----|
| 1 | 库存平衡 | 所有物品库存天数相同 |
| 2 | 非负装载 | 装载量 ≥ 0 |
| 3 | 不超容量 | 重量 ≤ 2000t 且 体积 ≤ 2000m³ |
| 4 | 填充率最大化 | 填充率尽可能接近100% |

---

## 六、技术约束

| 约束项 | 要求 |
|:-------|:-----|
| 计算精度 | 平衡天数保留2位小数 |
| 物品数量 | 支持任意种类物品扩展 |
| 运行时间 | < 1秒 |
