这是一个通用的多维装箱优化模型 (Multi-Dimensional Bin Packing Optimization Model)，专门针对“单次运输综合利用率最大化”的目标进行构建。该模型不依赖具体数据，适用于任何具有重量和体积双重约束的物流场景。

问题定义与符号系统 (Notation System)

1.1 集合 (Sets)
I = {1, 2, dots, n}: 货物种类集合，n 为物品种类数。
K = {1, 2, dots, M}: 可用运输航次（箱子）集合，M 为一个足够大的上界（例如总货物体积/单船体积）。
S_k subseteq I: 第 k 次运输中被选中的货物子集。

1.2 参数 (Parameters)
D_i: 第 i 种货物的总需求量 (Total Demand)。
w_i: 第 i 种货物的单位重量。
v_i: 第 i 种货物的单位体积。
W^{cap}: 单个运输单元的最大载重限制。
V^{cap}: 单个运输单元的最大容积限制。
alpha, beta: 权重系数，分别代表对重量利用率和体积利用率的重视程度（通常取 alpha=beta=0.5 表示同等重要，或根据成本动态调整）。

1.3 决策变量 (Decision Variables)
x_{i,k} in mathbb{Z}_{ge 0}: 第 k 次运输中装载第 i 种货物的数量。
y_k in {0, 1}: 二进制变量。若第 k 次运输被启用（即装载量 >0），则 y_k=1，否则为 0。
u^w_k in [0, 1]: 第 k 次运输的重量利用率。
u^v_k in [0, 1]: 第 k 次运输的体积利用率。

数学模型构建 (Mathematical Formulation)

本模型采用两阶段优化策略 (Two-Stage Optimization) 或 分层目标函数，以严格保证“单次最大化”的逻辑：
第一阶段：确定最小运输次数 m^*。
第二阶段：在固定次数 m^* 下，按顺序最大化每一次的利用率得分。

2.1 基础约束 (Constraints)

(1) 需求满足约束
所有航次的装载量之和必须等于总需求量：
 sum_{k=1}^{M} x_{i,k} = D_i, quad forall i in I 

(2) 容量限制约束
每次运输的重量和体积不得超过上限，且仅当航次启用时才生效：
 sum_{i=1}^{n} w_i x_{i,k} le W^{cap} cdot y_k, quad forall k in K 
 sum_{i=1}^{n} v_i x_{i,k} le V^{cap} cdot y_k, quad forall k in K 

(3) 利用率定义约束
 u^w_k = frac{sum_{i=1}^{n} w_i x_{i,k}}{W^{cap}}, quad forall k in K 
 u^v_k = frac{sum_{i=1}^{n} v_i x_{i,k}}{V^{cap}}, quad forall k in K 

(4) 逻辑关联约束
如果 x_{i,k} > 0，则必须 y_k=1（通常由容量约束隐含，但在某些求解器中需显式声明）：
 sum_{i=1}^{n} x_{i,k} le text{BigM} cdot y_k 

目标函数设计 (Objective Functions)

为了实现“单次最大化”，我们不能简单地求总和最大（那样可能导致某次 100%，另一次 10%），而应采用序贯贪婪优化 (Sequential Greedy Optimization) 或 极小化方差 + 极大化均值 的组合。

方案 A：序贯最大化模型 (推荐，符合“单次极致”逻辑)

此方案通过递归方式，确保第 1 次达到理论极限，然后在剩余货物中让第 2 次达到极限，以此类推。

步骤 1：确定最小航次数 m^*
 min sum_{k=1}^{M} y_k s.t. 基础约束 (1)-(2)
得到最优值 m^*。

**步骤 2：构建第 k 次运输的局部优化模型 (k = 1, dots, m^*)**
对于当前航次 k，在剩余货物集合 I_{rem} 中求解：
 text{Maximize } Z_k = alpha cdot u^w_k + beta cdot u^v_k s.t.
sum_{i in I_{rem}} w_i x_{i,k} le W^{cap}
sum_{i in I_{rem}} v_i x_{i,k} le V^{cap}
x_{i,k} le D_i^{rem} (剩余可用量)
x_{i,k} in mathbb{Z}_{ge 0}
注：求解完 Z_k 后，更新剩余需求量 D_i^{rem} leftarrow D_i^{rem} - x_{i,k}，然后进入 k+1 次循环。

方案 B：全局均衡最大化模型 (备选，追求整体平滑)
如果希望避免某次特别低，可以引入惩罚项，但不推荐用于你描述的“构造完美航次”场景，因为它会牺牲第一趟的完美度去填补最后一趟的空缺。

 text{Maximize } Z = sum_{k=1}^{m^*} (alpha u^w_k + beta u^v_k) - lambda cdot text{Var}(u_k) 
其中 text{Var}(u_k) 是各次利用率的方差，lambda 是惩罚系数。

关键特性分析：密度互补原理 (Density Complementarity)

该模型的核心数学本质在于寻找密度互补对。

定义货物 i 的密度为 rho_i = w_i / v_i。
目标航次的目标密度为 rho^* = W^{cap} / V^{cap}。

若要使 u^w_k approx 1 且 u^v_k approx 1，则所选货物子集的加权平均密度必须接近 rho^*：
 frac{sum w_i x_{i,k}}{sum v_i x_{i,k}} approx rho^* 

数学推论：
若所有可用货物的 rho_i > rho^*，则体积先满，重量利用率必然  rho^* 和 rho_{low} < rho^* 的货物，且数量比例适当，才能构造出 Z_k approx alpha + beta 的完美解。

通用求解算法流程 (Algorithm Flow)

基于上述模型，通用的计算步骤如下：

数据预处理：
    计算每种货物的密度 rho_i。
    计算系统目标密度 rho^* = W^{cap} / V^{cap}。
    计算理论下限 m^* = max(lceil sum w_i D_i / W^{cap} rceil, lceil sum v_i D_i / V^{cap} rceil)。

**迭代优化 (Loop k from 1 to m^*)**：
    输入：当前剩余货物列表 { (D_i^{rem}, w_i, v_i) }。
    构建整数规划 (IP) 子问题：
         max alpha frac{sum w_i x_i}{W^{cap}} + beta frac{sum v_i x_i}{V^{cap}} 
         text{s.t. } sum w_i x_i le W^{cap}, quad sum v_i x_i le V^{cap}, quad 0 le x_i le D_i^{rem} 
    求解：使用求解器 (如 Gurobi/CBC) 得到最优解 x^*_{i,k}。
    更新：D_i^{rem} leftarrow D_i^{rem} - x^*_{i,k}。
    记录：保存该航次的装载方案和利用率得分。

输出：
    各航次详细清单。
    各航次 u^w_k, u^v_k 及综合得分 Z_k。

模型扩展性 (Scalability)

此模型可轻松扩展至更多维度：
增加维度：若还有“价值”、“危险性”或“特殊舱位”限制，只需增加约束 sum p_i x_{i,k} le P^{cap} 并在目标函数中加入相应项。
多船型：若船队中有不同规格的船 (W^{cap}_j, V^{cap}_j)，可将 k 索引扩展为 (k, j)，并增加选择船型的二进制变量。
时间窗口：若货物有截止时间，可增加时间索引 t，转化为时空网络流问题。
